IS 670 Lab 3: Decision Tree¶

This data set contains information of cars purchased at the Auction.
We will use this file to predict the quality of buying decisions and visualize decision processes.

VARIABLE DESCRIPTIONS:
Auction: Auction provider at which the vehicle was purchased
Color: Vehicle Color
IsBadBuy: Identifies if the kicked vehicle was an avoidable purchase
MMRCurrentAuctionAveragePrice: Acquisition price for this vehicle in average condition as of current day
Size: The size category of the vehicle (Compact, SUV, etc.)
TopThreeAmericanName:Identifies if the manufacturer is one of the top three American manufacturers
VehBCost: Acquisition cost paid for the vehicle at time of purchase
VehicleAge: The Years elapsed since the manufacturer's year
VehOdo: The vehicles odometer reading
WarrantyCost: Warranty price (term=36month and millage=36K)
WheelType: The vehicle wheel type description (Alloy, Covers)

Target variable: IsBadBuy

Q1. Find one path in the tree to a leaf node that is classified to IsBadBuy = 'Yes'. What is this path/rule's misclassification error rate?

import numpy as np
from sklearn.tree import _tree

# --- Find leaf nodes that predict "Yes" ---
tree_ = model.tree_
classes = model.classes_
yes_index = list(classes).index('Yes')

leaf_ids = np.where((tree_.children_left == _tree.TREE_LEAF) &
                    (tree_.children_right == _tree.TREE_LEAF))[0]

yes_leaves = []
for node_id in leaf_ids:
    counts = tree_.value[node_id][0]   # class counts at this leaf
    pred_class = classes[np.argmax(counts)]
    if pred_class == 'Yes':
        yes_leaves.append(node_id)

print("Number of leaves predicting 'Yes':", len(yes_leaves))
leaf = yes_leaves[0]   # pick the first "Yes" leaf (any one is OK)

# --- Compute exact leaf misclassification error rate ---
counts = tree_.value[leaf][0]
n_total = counts.sum()
n_yes = counts[yes_index]
leaf_error = 1 - (n_yes / n_total)

print("Chosen leaf node:", leaf)
print("Class counts [No, Yes] (order = model.classes_):", counts)
print(f"Misclassification error rate at this leaf = {leaf_error:.6f}")

# --- Print the exact rule/path to that leaf ---
# Build the rule by walking from root to leaf using decision path on training data
# We'll find one training sample that ends up in this leaf and trace its decisions.

X_train = predictors_train.values
node_indicator = model.decision_path(X_train)
leaf_id_train = model.apply(X_train)

sample_id = np.where(leaf_id_train == leaf)[0][0]
feature = tree_.feature
threshold = tree_.threshold
feature_names = predictors_train.columns

node_index = node_indicator.indices[
    node_indicator.indptr[sample_id]: node_indicator.indptr[sample_id + 1]
]

print("\nRule path to the 'Yes' leaf:")
for node_id in node_index:
    if node_id == leaf:
        continue
    fname = feature_names[feature[node_id]]
    thresh = threshold[node_id]
    # check direction taken by this sample
    if X_train[sample_id, feature[node_id]] <= thresh:
        print(f" - {fname} <= {thresh:.6f}")
    else:
        print(f" - {fname} >  {thresh:.6f}")
print(" => Predict: Yes")

Q5. Build a decision tree model using training data, setting the maximum depth to 3, and assess its performance against the earlier model. Discuss which model you would favor and why.

The depth-3 decision tree performs slightly better than the depth-2 model. While both models achieve the same overall accuracy of 63%, the depth-3 model improves recall for the minority class (IsBadBuy = “Yes”) from 59% to 61%, meaning it correctly identifies more bad-buy vehicles. Additionally, the AUC increases from 0.6395 (depth-2) to 0.6464 (depth-3), indicating better overall classification performance.

Since the depth-3 model improves minority class detection and has a higher AUC without reducing overall accuracy, I would favor the depth-3 model. The slight increase in complexity is justified by the improvement in predictive performance.

Q6. Experiment with the splitting criteria “gini”, “entropy”, and “log_loss”, and vary max_depth from 2 to 30. Which configuration produces the highest AUC?

After testing decision trees with different criteria (gini, entropy, and log_loss) and maximum depths ranging from 2 to 30, the best-performing model achieved an AUC of 0.657346 using:

Criterion: entropy

Max depth: 5

The log_loss criterion with max_depth = 5 produced the same AUC (0.657346), and gini with max_depth = 5 performed almost identically (0.657340). However, entropy with max_depth = 5 is selected as the best configuration since it achieves the highest AUC.

Compared to the earlier depth-2 model (AUC = 0.6395) and depth-3 model (AUC = 0.6464), the depth-5 model shows a noticeable improvement in predictive performance. This indicates that allowing slightly deeper trees helps the model better capture patterns in the data without severe overfitting.

Therefore, the preferred model is the decision tree with criterion = entropy and max_depth = 5, as it provides the highest overall classification performance based on AUC.

import pandas as pd
from sklearn.tree import DecisionTreeClassifier
from sklearn.metrics import roc_auc_score

criteria = ["gini", "entropy", "log_loss"]
depths = range(2, 31)

y_test_bin = target_test.map({"No": 0, "Yes": 1})

rows = []

for crit in criteria:
    for d in depths:
        m = DecisionTreeClassifier(criterion=crit, random_state=1, max_depth=d)
        m.fit(predictors_train, target_train)

        yes_idx = list(m.classes_).index("Yes")
        proba_yes = m.predict_proba(predictors_test)[:, yes_idx]
        auc = roc_auc_score(y_test_bin, proba_yes)

        rows.append({"criterion": crit, "max_depth": d, "AUC": auc})

df_results = pd.DataFrame(rows).sort_values("AUC", ascending=False)

print("Top 10 configurations by AUC:")
print(df_results.head(10).to_string(index=False))

best = df_results.iloc[0]
print("\nBest configuration:")
print(best.to_string())

# (Optional) best per criterion
print("\nBest per criterion:")
print(df_results.sort_values("AUC", ascending=False).groupby("criterion").head(1).to_string(index=False))
