IS 670 Lab 4: Decision Tree vs Naive Bayes¶

This data set contains information of cars purchased at the Auction.
We will use this file to predict the quality of buying decisions and visualize decision processes.

VARIABLE DESCRIPTIONS:
Auction: Auction provider at which the vehicle was purchased
Color: Vehicle Color
IsBadBuy: Identifies if the kicked vehicle was an avoidable purchase
MMRCurrentAuctionAveragePrice: Acquisition price for this vehicle in average condition as of current day
Size: The size category of the vehicle (Compact, SUV, etc.)
TopThreeAmericanName:Identifies if the manufacturer is one of the top three American manufacturers
VehBCost: Acquisition cost paid for the vehicle at time of purchase
VehicleAge: The Years elapsed since the manufacturer's year
VehOdo: The vehicles odometer reading
WarrantyCost: Warranty price (term=36month and millage=36K)
WheelType: The vehicle wheel type description (Alloy, Covers)

Target variable: IsBadBuy

2. Partition the data set for Decision Tree model¶

In [53]:
# 1. Use the cleaned dataframe (carAuction) which has "Yes/No" labels
# Create dummy variables - this will create 'IsBadBuy_Yes'
car_kick_dummies = pd.get_dummies(carAuction, drop_first=True)

# 2. Define the target and predictors
# Now that we have dummy variables, the target column is 'IsBadBuy_Yes'
target_col = 'IsBadBuy_Yes'
predictors = car_kick_dummies.drop(target_col, axis=1)
target = car_kick_dummies[target_col]

# 3. Partition the data (70% Train, 30% Test)
predictors_train, predictors_test, target_train, target_test = train_test_split(
    predictors, target, test_size=0.3, random_state=1, stratify=target
)

# 4. Balance the training data using Undersampling
# Combine train predictors and target for sampling
combined_train_df = pd.concat([predictors_train, target_train], axis=1)

# Separate Majority (0 = No) and Minority (1 = Yes)
majority_df = combined_train_df[combined_train_df[target_col] == 0]
minority_df = combined_train_df[combined_train_df[target_col] == 1]

# Randomly sample the majority to match the minority count
undersampled_majority = majority_df.sample(n=len(minority_df), random_state=1)
balanced_data = pd.concat([undersampled_majority, minority_df]).sample(frac=1, random_state=1)

# Final balanced training sets
predictors_train = balanced_data.drop(columns=[target_col])
target_train = balanced_data[target_col]

print("Balanced Training Set Size:", predictors_train.shape)
print("Target proportions in balanced training set:")
print(target_train.value_counts(normalize=True))

Answer Q1 to Q5 based on the NB model.

Q1. On the testing set, how many Yes (bad buy) cars are predicted as No (not bad buy)? Answer: 803

This is the count of False Negatives (bottom-left square). These are cars that were actually "Bad Buys" but the model incorrectly flagged them as safe to buy.

Q2. On the testing set, how many cars are predicted as Not bad buy?Answer: 11,860This is the sum of the entire "No" column under "Predicted label".Calculation: $11,057$ (True Negatives) $+ 803$ (False Negatives) $= 11,860$.

Q3. On the testing set, how many bad buy cars are identified correctly? Answer: 1,122

This is the count of True Positives (bottom-right square). These are the "Bad Buy" cars the model successfully caught.

Q4. If the model predicts a car as a bad buy, what is the probability that such prediction is correct? (Precision)Answer: ~13.51%This is the Precision for the "Yes" class.Calculation: $\frac{True Positives}{Total Predicted Yes} = \frac{1,122}{1,122 + 7,182} = \frac{1,122}{8,304} \approx 0.1351$.

Q5. What percentage of kick cars can be detected by this model? (Recall)Answer: ~58.29%This is the Recall for the "Yes" class.Calculation: $\frac{True Positives}{Total Actual Yes} = \frac{1,122}{1,122 + 803} = \frac{1,122}{1,925} \approx 0.5829$.

Q6. Compare the two models. The Decision Tree usually provides better precision but might struggle to catch as many "Bad Buys" if the depth is shallow.

The Naive Bayes model here has a decent recall (~58%) but very low precision (~13.5%), meaning it flags a lot of "False Alarms" (7,182 good cars marked as bad buys).Q6. Compare the two models. Q8. Why should one get the models' performance on the training set when we have the testing set?

Checking the training set allows you to detect overfitting.

If a model performs perfectly on the training data but poorly on the testing data, it has "memorized" the specific rows instead of learning general patterns.
