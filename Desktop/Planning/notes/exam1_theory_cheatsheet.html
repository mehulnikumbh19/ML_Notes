1. MACHINE LEARNING FOR BUSINESS ANALYTICS

  • What is ML? Using data + algorithms to build models that predict or classify outcomes so a business can act (e.g., predict bad buys at auction, predict customer churn, classify spam).
  • Supervised learning (this course): We have labeled data — each record has a known outcome (the target). The model learns patterns from the features (predictors) to predict the target on new, unseen data.
  • Classification vs Regression: Classification predicts a category (e.g., Good Buy vs Kick, True vs False). Regression predicts a number (not covered in Exam 1).
  • Why does it matter? Better predictions → better business decisions → reduced losses / increased revenue.

3. DATA TYPES & STRUCTURES

Variable Types

  • Numeric (quantitative): Continuous (e.g., VehBCost, Attack) or discrete (e.g., VehicleAge). Summarize with mean, median, std; visualize with histograms, boxplots, scatter plots.
  • Categorical (qualitative): Nominal (no order — e.g., Color, Type1, Auction) or ordinal (ordered — e.g., Size: Small < Medium < Large). Summarize with counts/proportions; visualize with count plots, bar charts.

Data Structures

  • Tabular data: Rows = records/observations (one per entity). Columns = variables/features (one per attribute).
  • DataFrame: pandas 2D table with column names and row index. Created from CSV via pd.read_csv().
  • Series: A single column extracted from a DataFrame — one-dimensional.
  • CSV: Comma-separated values — a text file where each line is a row, commas separate columns.

4. DATA UNDERSTANDING & EXPLORATION

Why Explore First?

Garbage in → garbage out. You need to know variable types, distributions, missing values, outliers, and relationships before choosing a model. Exploration also reveals class imbalance (e.g., only ~9.55% kicks), which affects how you evaluate.

Missing Values

  • What: NaN / null — no recorded value for that cell. Could mean "not applicable" (e.g., Type2 for single-type Pokemon) or data-entry error.
  • Why handle: Most sklearn models cannot accept NaN. Leaving them unhandled causes errors or biased results.
  • Strategies: (a) Drop rows/columns with too many missing. (b) Impute numeric: fill with median (robust to outliers) or mean. (c) Impute categorical: fill with mode (most frequent value) or treat missing as its own category (e.g., "None").
  • Trap: Filling a numeric column with a string (e.g., 'None') converts its data type to object — breaks numeric operations.

Outliers

  • What: Values far from the rest of the data. Boxplots show them as points beyond the whiskers.
  • IQR method: Q1 = 25th percentile, Q3 = 75th percentile, IQR = Q3 − Q1. Lower fence = Q1 − 1.5×IQR, Upper fence = Q3 + 1.5×IQR. Points outside = outliers.
  • Handling: Cap/clip to fences, remove, or leave (depends on domain). In Assignment 1, IQR method used for WarrantyCost.

Class Imbalance

  • What: One class is much more frequent than the other (e.g., 90% good buys, 10% kicks).
  • Problem: A model that always predicts the majority class gets high accuracy but zero recall for the minority class — useless.
  • Solution: Use precision, recall, F1 instead of accuracy alone. Evaluate minority-class performance explicitly.

5. PREPROCESSING FOR ML

One-Hot Encoding

  • Why: sklearn models require numeric inputs. Categorical columns (text) must be converted to numbers.
  • How: Each category becomes a new 0/1 column. E.g., Color = {Red, Blue, Green} → Color_Blue (0/1), Color_Green (0/1). Red is inferred when both are 0.
  • drop_first: Drops one category per column to avoid multicollinearity — when one column can be perfectly predicted from the others. Keeps the model mathematically stable.

Feature Scaling / Normalization

  • Why: Distance-based algorithms (KNN) are affected by the scale of features. If VehOdo ranges 0–150,000 and VehicleAge ranges 0–10, VehOdo dominates the distance calculation.
  • MinMaxScaler: Transforms each feature to [0, 1] range. Formula: x_scaled = (x − min) / (max − min).
  • StandardScaler: Transforms to mean = 0, std = 1 (z-score normalization).
  • Critical rule: Fit the scaler on training data only, then transform both train and test. Fitting on test = data leakage (test info leaks into preprocessing).
  • Which models need scaling? KNN: always. Decision Tree: no (splits are threshold-based, scale-invariant). Naive Bayes: no.

Target & Predictors

  • Target (y): The variable we want to predict (e.g., Class, IsBadBuy, Legendary).
  • Predictors / Features (X): All other variables the model uses as input to make predictions.
  • Always separate y from X before train/test split. Never include the target in X.

Train / Test Split

  • Purpose: Simulate how the model will perform on new, unseen data. Fit the model on training set; evaluate on test set.
  • Typical ratio: 70% train / 30% test (course standard).
  • random_state: A seed number that makes the random split reproducible — same split every time.
  • Rule: Never evaluate only on training data — this gives an overly optimistic estimate. Never tune hyperparameters using the test set — use a validation set or cross-validation instead.

6. DECISION TREES

How It Works

  • A flowchart of yes/no questions on features. Each internal node tests a feature (e.g., "Is VehicleAge ≤ 4.5?"). Each branch follows the answer. Each leaf assigns a predicted class.
  • The tree is built top-down: at each node, the algorithm finds the feature and threshold that best separates the classes (maximizes "purity" of child nodes).
  • Interpretable: You can read the tree like a set of if-then rules; you can see exactly which features drive predictions — valuable for business explanations.

Entropy & Information Gain

  • Entropy measures impurity (disorder) in a node:
H = −Σ pᵢ log₂(pᵢ)   where pᵢ = proportion of class i in the node
  • Entropy = 0 → node is pure (all one class). Maximum entropy → node is maximally mixed (50/50 for binary).
  • Information Gain = entropy(parent) − weighted average entropy(children). The algorithm picks the split with the highest information gain (biggest entropy reduction).

Gini Impurity (alternative)

  • Gini = 1 − Σ pᵢ². Also measures impurity; 0 = pure. sklearn default is gini; course labs use entropy.
  • Both produce similar trees in practice. Entropy uses log; Gini is computationally simpler.

max_depth (Regularization)

  • Limits how deep the tree can grow. Smaller max_depth = simpler tree = less overfitting = better generalization (but may underfit if too small). No limit = tree can memorize training data (overfit).
  • Assignment 1 tested max_depth = 3, 5, 7, 10, unlimited to compare overfitting vs generalization.

Strengths & Weaknesses

StrengthsWeaknesses
Highly interpretable (visual rules)Prone to overfitting if unconstrained
No scaling neededCan be unstable (small data changes → different tree)
Handles both numeric & categorical (after encoding)Greedy algorithm — locally optimal splits, not globally

7. NAIVE BAYES

Bayes' Theorem

The foundation of the algorithm. For classification:

P(Class | features) ∝ P(features | Class) × P(Class)
  • Prior — P(Class): How common is each class in the training data? (e.g., 90% good buy, 10% kick). Represents our belief before seeing the features.
  • Likelihood — P(features | Class): Given a class, how likely are these feature values? Modeled differently by each NB variant.
  • Posterior — P(Class | features): What we want — the probability of each class given the observed features. We predict the class with the highest posterior.

The "Naive" Assumption

  • Assumes all features are conditionally independent given the class. This means:
P(x₁, x₂, …, xₙ | Class) ≈ P(x₁ | Class) × P(x₂ | Class) × … × P(xₙ | Class)
  • This is almost never true in real data (e.g., Attack and Defense in Pokemon are correlated). But the simplification makes the model fast, tractable, and surprisingly effective in practice.
  • "Naive" refers specifically to this independence assumption — not that the model is simplistic.

Variants

VariantFeature TypeAssumption
GaussianNBContinuous numericEach feature follows a normal (Gaussian) distribution per class
MultinomialNBDiscrete counts (non-negative)Features are word/event counts (e.g., word frequencies in text)
BernoulliNBBinary (0/1)Each feature is present or absent
  • Lab 4 uses MultinomialNB on one-hot encoded car data. For continuous features with possible negatives, use GaussianNB.

Typical Trade-off (NB vs DT)

  • NB tends to have higher recall (catches more positives) but lower precision (more false alarms).
  • DT tends to have higher precision but may miss more positives (lower recall).
  • Choice depends on the business cost: is a false positive (false alarm) or a false negative (missed bad buy) more expensive?

8. K-NEAREST NEIGHBORS (KNN)

How It Works

  • For a new point to classify: (1) Calculate distance to every training point. (2) Find the K closest (nearest neighbors). (3) Take a majority vote — the most common class among those K neighbors is the prediction.
  • Lazy learner / instance-based: No model is built during training. All computation happens at prediction time (looks up neighbors on the fly). Contrast with eager learners (DT, NB) that build a model at fit time.

Distance Metric

Euclidean distance: d(x, y) = √( Σ (xᵢ − yᵢ)² )
  • Default in sklearn. Other options include Manhattan distance (|xᵢ − yᵢ|). The choice of distance metric matters but Euclidean is standard for this course.
  • Scaling is mandatory because features with larger ranges dominate the distance. E.g., VehOdo (0–150k) would completely overshadow VehicleAge (0–10) without scaling.

Choosing K

K valueEffectRisk
K = 1Each point's nearest neighbor is itself → 100% train accuracySevere overfitting — memorizes noise; poor test accuracy
Small K (e.g., 3–5)Sensitive to local patterns; complex decision boundarySome overfitting risk
Moderate K (e.g., 4–20)Smoother boundary; balances local detail and noiseSweet spot; Lab 5 prefers K=4
Very large KBoundary approaches majority class everywhereUnderfitting — too smooth; ignores local structure

Strengths & Weaknesses

StrengthsWeaknesses

9. CLASSIFICATION METRICS

Confusion Matrix

A 2×2 table comparing actual vs predicted labels. Foundation for all other metrics:

Predicted Negative (0)Predicted Positive (1)
Actual Negative (0)TN — True Negative (correctly predicted negative)FP — False Positive (predicted positive, actually negative; "false alarm")
Actual Positive (1)FN — False Negative (predicted negative, actually positive; "miss")TP — True Positive (correctly predicted positive; "hit")

Metric Formulas & Meaning

MetricFormulaInterpretationWhen to prioritize
Accuracy(TP + TN) / total% of all predictions that are correctBalanced classes only; misleading when imbalanced
PrecisionTP / (TP + FP)Of those predicted positive, what fraction actually is positive?When false positives are costly (e.g., spam filter: don't block real email)
Recall (Sensitivity)TP / (TP + FN)Of all actual positives, what fraction did we catch?When missing positives is costly (e.g., Don't Get Kicked: don't miss bad buys)
F1 Score2 × P × R / (P + R)Harmonic mean of precision and recall; balances bothWhen you need a single metric that considers both P and R
AUCArea under the ROC curveMeasures ranking ability: how well the model separates classes across all thresholds. 0.5 = random; 1.0 = perfectThreshold-independent; good for comparing models overall

ROC Curve

  • Plots True Positive Rate (Recall) vs False Positive Rate (FP / (FP + TN)) at every possible decision threshold.
  • A model that perfectly separates classes has a curve that hugs the top-left corner (AUC = 1.0). A random model traces the diagonal (AUC = 0.5).
  • AUC is useful because it's threshold-independent — summarizes performance across all cutoff points.

Why Accuracy Alone Fails (Imbalanced Data)

Example: 90% good buys, 10% kicks. A model that predicts "good buy" for everything gets 90% accuracy — but 0% recall for kicks. It catches zero bad buys. Precision and recall for the minority class reveal this failure; accuracy hides it.

10. OVERFITTING & GENERALIZATION

Definitions

  • Overfitting: Model learns the training data too well — captures noise and idiosyncrasies, not just real patterns. Result: excellent training performance, poor test performance. The model fails to generalize.
  • Underfitting: Model is too simple to capture real patterns. Result: poor performance on both training and test data.
  • Generalization: The ability of a model to perform well on new, unseen data (the test set). This is the ultimate goal of ML.

How to Detect

PatternDiagnosisExample from Labs
High train accuracy, low test accuracy (large gap)OverfittingKNN K=1: 100% train, ~55% test; DT no max_depth: 100% train, ~55% test
Low train accuracy, low test accuracyUnderfittingKNN very large K; DT max_depth=1
Reasonably high test accuracy, small train-test gapGood generalizationKNN K=4: ~72% train, ~67% test; DT max_depth=3: ~75% train, ~72% test

Bias–Variance Trade-off

  • Bias: Error from overly simple assumptions. High bias → underfitting (e.g., very large K, very shallow tree).
  • Variance: Error from sensitivity to small fluctuations in training data. High variance → overfitting (e.g., K=1, very deep tree).
  • The goal is to find the sweet spot — a model complex enough to capture real patterns but simple enough not to memorize noise.

Regularization (How to Fight Overfitting)

ModelRegularization ParameterHow It Helps
Decision Treemax_depthSmaller depth = simpler tree = less overfitting
KNNK (n_neighbors)Larger K = smoother boundary = less overfitting
Naive Bayes(few tunable params)Inherently simple; rarely overfits

Key Rules

  • Always evaluate on the test set to estimate real-world performance. Training accuracy alone is meaningless.
  • Never tune hyperparameters on the test set. Use a separate validation set or cross-validation.
  • More complex ≠ better. If two models have similar test accuracy, prefer the simpler one.
  • 100% training accuracy is a red flag for overfitting (especially KNN K=1, unconstrained DT).

When to Choose Which

  • Need to explain decisions to business stakeholders? → Decision Tree (readable rules).
  • Fast baseline with limited data? → Naive Bayes (fast, few params, robust).
  • Numeric features, irregular decision boundaries? → KNN (after scaling).
  • Imbalanced data? → Any model, but evaluate with precision/recall/F1, not accuracy.
  • Suspect overfitting? → DT: reduce max_depth. KNN: increase K. NB: usually fine.
