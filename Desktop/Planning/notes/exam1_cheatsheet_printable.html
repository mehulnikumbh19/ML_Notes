DATA & PREPROCESSING

  • Load: df = pd.read_csv('path.csv')
  • Inspect: df.shape, df.head(), df.tail(), df.columns
  • Missing: df.isnull().sum() → count per column
  • Fill (categorical): df = df.fillna('None') — NOT for numeric cols
  • One-hot encode: pd.get_dummies(df, columns=[...], drop_first=True, dtype=int)
  • Define X, y: X = df.drop('Target', axis=1); y = df['Target']

TRAIN / TEST SPLIT

X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=0)
  • Fit on train; evaluate on test only → measures generalization
  • Never evaluate only on training data

DECISION TREE

  • Idea: Flowchart of yes/no questions on features; leaves = predicted class
  • Entropy: H = −Σ pᵢ log₂(pᵢ) — measures impurity; 0 = pure, max = mixed
  • Tree picks splits that reduce entropy most (maximize information gain)
  • max_depth: Limits tree depth; smaller = less overfitting
  • criterion: 'entropy' or 'gini'
model = DecisionTreeClassifier(criterion='entropy',
    max_depth=3, random_state=1)
model.fit(X_train, y_train)
y_pred = model.predict(X_test)

NAIVE BAYES

  • Bayes: P(class|features) ∝ P(class) × P(f₁|class) × P(f₂|class) × …
  • "Naive" = assumes features are independent given class
  • Fast; few parameters; no scaling needed
  • GaussianNB for continuous; MultinomialNB for counts
from sklearn.naive_bayes import GaussianNB
model = GaussianNB()
model.fit(X_train, y_train)
y_pred = model.predict(X_test)

KNN (K-NEAREST NEIGHBORS)

  • Idea: Find K closest training points by distance → majority vote
  • MUST SCALE features first (distance-based)
  • Small K (e.g. 1) → overfitting; Large K → smoother boundary
  • Odd K avoids ties; too large K → underfitting
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
X_train_s = scaler.fit_transform(X_train)
X_test_s = scaler.transform(X_test)  # transform only!

from sklearn.neighbors import KNeighborsClassifier
model = KNeighborsClassifier(n_neighbors=5)
model.fit(X_train_s, y_train)
y_pred = model.predict(X_test_s)

OVERFITTING vs GENERALIZATION

  • Overfitting: Great on train, poor on test (memorizing noise)
  • Generalization: Good on unseen (test) data
  • Fix DT: Reduce max_depth / prune
  • Fix KNN: Increase K (smooths boundary)
  • Very deep tree or K=1 → overfitting

TOP PITFALLS

  • Accuracy alone + imbalanced data = misleading
  • KNN without scaling = wrong distances
  • Evaluate on train instead of test = overly optimistic
  • fillna('None') on numeric col = converts to object
  • confusion_matrix(y_true, y_pred) — actual first, predicted second
  • Categorical cols must be encoded before sklearn models
