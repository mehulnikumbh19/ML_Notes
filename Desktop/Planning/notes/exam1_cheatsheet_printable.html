6. DECISION TREE

Concepts

  • Model that splits on features (e.g. VehicleAge ≤ 4.5 → left, else right). Each node = question; leaves = predicted class. Interpretable — can see which features drive predictions.
  • Entropy: H = −Σ pᵢ log₂(pᵢ) over classes. Lower = purer (one class dominates). Leaf with one class has entropy 0. Tree picks splits that reduce entropy most = maximize information gain.
  • Gini (alternative): G = 1 − Σ pᵢ². Also measures impurity.
  • criterion: 'entropy', 'gini', or 'log_loss'. Labs use entropy. max_depth: Limits depth; smaller = simpler = less overfitting.

Code (all variations)

  • DecisionTreeClassifier(criterion='entropy', random_state=1, max_depth=2) [Lab3]; max_depth=3 [Lab4]. DecisionTreeClassifier(max_depth=5, random_state=42) [Assign1 — default gini].
  • DecisionTreeClassifier() with no max_depth = unconstrained (overfits, 100% train). model.fit(X_train, y_train); y_pred = model.predict(X_test).
  • Plot tree: from sklearn.tree import plot_tree; plot_tree(model, feature_names=X_train.columns.tolist(), max_depth=3, filled=True, rounded=True). export_text(model, feature_names=list(X.columns)) — text version.

7. NAIVE BAYES

Concepts

  • Bayes: P(Class|X) ∝ P(X|Class) × P(Class). Predict class that maximizes posterior. Prior = P(Class); Likelihood = P(X|Class).
  • "Naive": P(X|Class) ≈ P(x₁|Class)×P(x₂|Class)×… — features conditionally independent given class. Simplifies computation; often works well despite being false.
  • Fast; no scaling needed; few parameters. Trade-off vs DT: NB often higher recall, lower precision (more false alarms); DT often higher precision.

sklearn variants

  • GaussianNB — continuous numeric; assumes normal distribution per class.
  • MultinomialNB — discrete/count features (non-negative). Lab 4 car data. Negative/continuous → use GaussianNB or scale first.
  • BernoulliNB — binary 0/1 features.
  • model = MultinomialNB() or GaussianNB(); model.fit(X_train, y_train); y_pred = model.predict(X_test).

8. KNN (K-NEAREST NEIGHBORS)

Concepts

  • For new point, find K closest training points (by distance); majority vote = prediction. Lazy learner — no model at fit; all work at predict.
  • Euclidean distance: d(x,y) = √(Σ(xᵢ−yᵢ)²). Features with larger scale dominate → MUST scale (e.g. MinMaxScaler) so all features contribute fairly.
  • K=1: Nearest neighbor = self → 100% train accuracy, poor test (overfitting). Large K: Underfitting; predicts majority class. Sweet spot: moderate K (e.g. 4, 5). Curse of dimensionality: In high dimensions, KNN suffers; prefer NB or DT.

Scaling (critical)

  • MinMaxScaler: (x − min)/(max − min) per feature → [0, 1]. StandardScaler: z-score (mean 0, std 1).
  • scaler = MinMaxScaler(); X_train_s = scaler.fit_transform(X_train); X_test_s = scaler.transform(X_test). Fit on train only; transform both — fitting on test = data leakage.
  • After fit_transform, result is ndarray; restore names: X_train_s = pd.DataFrame(scaler.fit_transform(X_train)); X_train_s.columns = X_train.columns.
  • KNeighborsClassifier(n_neighbors=1) — overfit. n_neighbors=4 or 5 — better generalization.

Actual 1FN (missed)TP (correct pos)
  • cm = confusion_matrix(y_test, y_pred) — y_true first, y_pred second. ConfusionMatrixDisplay(confusion_matrix=cm, display_labels=model.classes_).plot() or ConfusionMatrixDisplay.from_predictions(y_test, y_pred). Custom labels: display_labels=['Good Buy','Kick'].

Formulas (memorize)

MetricFormulaWhen to use
Accuracy(TP+TN)/totalBaseline; misleading if imbalanced
PrecisionTP/(TP+FP)When FP costly (e.g. spam filter)
RecallTP/(TP+FN)When missing positives costly (e.g. bad buys)
F12×P×R/(P+R)Balance P and R

Code

  • print(classification_report(y_test, y_pred)) — P, R, F1 per class + accuracy. classification_report(y_test, y_pred, target_names=['Good Buy','Kick']).
  • accuracy_score(y_test, y_pred). precision_score(y_test, y_pred, pos_label=1, zero_division=0) — same for recall_score, f1_score. model.score(X_train, y_train) — in-sample accuracy.
  • AUC: Needs numeric y. If labels 'Yes'/'No': y_test_bin = y_test.map({'Yes':1,'No':0}). roc_auc_score(y_test, model.predict_proba(X_test)[:, 1]) — [:, 1] = probability of positive class. fpr, tpr, _ = roc_curve(y_test, model.predict_proba(X_test)[:, 1]) for plotting ROC.
  • Imbalanced data: Don't rely on accuracy; use precision, recall, F1 for minority class.

11. OVERFITTING & GENERALIZATION

  • Overfitting: Model fits training data too closely (captures noise); high train accuracy, low test. Examples: K=1 in KNN, very deep DT, unconstrained tree.
  • Underfitting: Model too simple; low train and low test. Examples: very large K, max_depth=1.
  • Generalization: Good performance on unseen (test) data. Goal of ML.
  • Bias–variance: Simple model = high bias, low variance; complex = low bias, high variance (overfitting).
  • Regularization: DT → limit max_depth; KNN → increase K. Never tune hyperparameters on test set; use validation or cross-validation.
  • Typical pattern: K=1 → 100% train, ~55% test; K=4 → ~72% train, ~67% test (better). Deep DT → 100% train, poor test; max_depth=3 → better test.

15. TOP PITFALLS & QUICK FIXES

  • fillna('None') on numeric → converts to object; use median/mode for numeric.
  • Accuracy alone when imbalanced → model predicting only majority can have high accuracy, 0 recall for minority.
  • KNN without scaling → distance dominated by one feature; scale first.
  • Evaluate only on train → overestimates performance; always report test metrics.
  • Fit scaler on test → data leakage; fit on train, transform both.
  • confusion_matrix(y_pred, y_test) wrong order → use (y_true, y_pred).
  • ValueError: could not convert string to float → encode categoricals (get_dummies).
  • FileNotFoundError → wrong path or Drive not mounted. NameError: pd not defined → run import cell first.
  • AUC needs probabilities: model.predict_proba(X_test)[:, 1] (positive class column).
