7. DECISION TREE

How it works

  • Splits on features (e.g. VehicleAge ≤ 4.5 → left/right); leaves assign class. [Lec3]
  • Entropy: H = −Σ pᵢ log₂(pᵢ). Lower = purer; 0 = single class. Split reduces entropy (max info gain). [Lec3]
  • criterion: 'entropy', 'gini', or 'log_loss'; Lab uses entropy. [Lab03,Lec3]
  • max_depth: Limits tree depth; smaller = less overfitting. [Lec3,Lec6]

Code — every variation

  • model = DecisionTreeClassifier(criterion='entropy', random_state=1, max_depth=2) [Lab03]

  • model = DecisionTreeClassifier() — no max_depth = unconstrained (overfits). [Lab05]
  • model.fit(X_train, y_train); y_pred = model.predict(X_test) [Lab03–05]

8. NAIVE BAYES

Concepts

  • Bayes: P(Class|X) ∝ P(X|Class) × P(Class). Pick class with highest posterior. [Lec4]
  • "Naive": P(X|Class) ≈ P(x₁|Class)×P(x₂|Class)×…; features conditionally independent given class. [Lec4]
  • Prior: P(Class); Likelihood: P(X|Class). [Lec4]

sklearn variants

  • GaussianNB: Continuous numeric; assumes normal per class. [Lec4,08]
  • MultinomialNB: Discrete/count features (non-negative). Lab 4 car data. [Lab04,Lec4]
  • BernoulliNB: Binary 0/1 features. [Lec4]
  • MultinomialNB + negative/continuous → use GaussianNB or scale first. [Lec4]

Code

  • model = MultinomialNB(); model.fit(X_train, y_train); y_pred = model.predict(X_test) [Lab04]
  • model = GaussianNB() for continuous. [08]
  • Typical trade-off: NB higher recall, lower precision (more false positives); DT higher precision. [Lec4]

9. KNN (K-NEAREST NEIGHBORS)

Concepts

  • Find K nearest training points (Euclidean distance); majority vote = prediction. [Lec5]
  • Euclidean: d(x,y) = √(Σ(xᵢ−yᵢ)²). Features with larger scale dominate → MUST SCALE. [Lec5]
  • K=1: Each point's nearest neighbor = itself → 100% train, poor test (overfitting). [Lab05,Lec5]
  • Large K: Underfitting; predicts majority class. [Lec5]
  • Lazy learner: No model at fit; work at predict. [Lec5]

Scaling — critical

  • scaler = MinMaxScaler(); X_train_s = scaler.fit_transform(X_train); X_test_s = scaler.transform(X_test) — fit on train only; transform both. [Lab05,08]
  • predictors_normalized = pd.DataFrame(min_max_scaler.fit_transform(predictors)); predictors_normalized.columns = predictors.columns — Preserve column names after fit_transform (returns ndarray). [Lab05]
  • from sklearn.preprocessing import MinMaxScaler or from sklearn import preprocessing; preprocessing.MinMaxScaler() [Lab05,08]
  • MinMax: maps to [0,1]; StandardScaler: mean 0, std 1. [Lec5]

Code

  • model = KNeighborsClassifier(n_neighbors=1) — overfitting. [Lab05]
  • model = KNeighborsClassifier(n_neighbors=4) or n_neighbors=5 — better generalization. [Lab05,08]

12. OVERFITTING & GENERALIZATION

  • Overfitting: High train, low test (memorizes noise). Examples: K=1, very deep DT. [Lec6]
  • Underfitting: Low train and test (too simple). Examples: large K, max_depth=1. [Lec6]
  • Generalization: Good on unseen (test) data. [Lec6]
  • Fix DT: Reduce max_depth. Fix KNN: Increase K. [Lec6]
  • Bias-variance: Simple = high bias; complex = high variance (overfitting). [Lec6]
  • Never tune on test; use validation or cross-validation. [Lec6]

15. TOP PITFALLS

  • fillna('None') on numeric → converts to object. [Lec2]
  • Accuracy alone when imbalanced → misleading. [Lec3]
  • KNN without scaling → wrong distances. [Lec5]
  • Evaluate only on train → no generalization estimate. [Lec6]
  • Fit scaler on test → data leakage. [Lab05]
  • confusion_matrix — y_true first, y_pred second. [Lab03]
  • Categorical cols not encoded → ValueError (string to float). [Lab03]
  • predict_proba(X_test)[:, 1] for AUC — positive class column. [Lab03]
