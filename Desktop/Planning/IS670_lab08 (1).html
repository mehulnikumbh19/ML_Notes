Part 1. SVM and Neural Network for Numeric Prediction¶


In order for a health insurance company to make money, it needs to collect more in yearly premiums than it spends on medical care to its beneficiaries. As a result, insurers invest a great deal of time and money in developing models that accurately forecast medical expenses for the insured population.

Medical expenses are difficult to estimate because the most costly conditions are rare and seemingly random. Still, some conditions are more prevalent for certain segments of the population. For instance, lung cancer is more likely among smokers than non-smokers, and heart disease may be more likely among the obese.

The goal of this analysis is to use patient data to estimate the average medical care expenses for such population segments. These estimates can be used to create actuarial tables that set the price of yearly premiums higher or lower, depending on the expected treatment costs.

The insurance data set has 1338 observations of 7 variables.
We will use this file to predict the medical expenses.

VARIABLE DESCRIPTIONS:
age: age in years
sex: gender
bmi: body mass index
children: how many children do they have?
smoker: do they smoke?
region: geographic region
expenses: yearly medical expenses

Target variable: expenses

Q1. Which C value provides the best performance? (0.5 points)

Answer: Based on the results, C = 100.0 performs best overall because it gives the lowest MAE. Although C = 10.0 has the lowest RMSE, the difference is small, while the MAE improvement at C = 100.0 is larger. Therefore, C = 100.0 is the best choice if we want the best general performance.

Q2. How does the cost parameter C impact SVM model performance? (0.5 points)

Answer: The cost parameter C controls the trade-off between maximizing the margin and minimizing training errors in an SVM model. A small C value creates a wider margin that tolerates more training errors, resulting in a simpler, more generalized model that may underfit. A large C value narrows the margin and penalizes misclassifications heavily, making the model fit the training data more tightly but potentially overfitting. In our experiments, increasing C from 1.0 to 100.0 steadily improved MAE and RMSE, suggesting the default C was underfitting. The key is finding the right C that captures underlying patterns without overfitting to noise.

Q3. Assume that you will lose each dollar your model's prediction misses due to an over-estimation or under-estimation. Which evaluation metric you should use? (0.5 points)

Answer: You should use Mean Absolute Error (MAE). MAE computes the average of the absolute differences between predicted and actual values, treating every dollar of error equally regardless of direction. Since the scenario states that you lose each dollar your prediction misses — meaning the cost is linearly proportional to the error magnitude — MAE is the most appropriate metric. Unlike RMSE, it does not disproportionately weight larger errors, making it ideal when every unit of deviation carries the same financial penalty.

Q4. Assume that the penalty for an erroneous prediction increases with the difference between the actual and predicted values. Which evaluation metric you should use? (0.5 points)

Answer: You should use Root Mean Squared Error (RMSE). RMSE squares individual errors before averaging and then takes the square root, which disproportionately penalizes larger prediction errors. In a scenario where the penalty grows with the magnitude of the error — meaning a $1,000 miss is far more costly than a $100 miss — RMSE effectively captures this non-linear, increasing penalty structure. By amplifying the impact of large deviations, RMSE ensures the model evaluation aligns with the assumption that bigger errors are progressively more expensive.

Q5. If you want to improve the MLP model's performance, what hyperparameters you should tune? (name at least 3)

Answer: To improve the MLP model's performance, the following hyperparameters should be tuned: (1) hidden_layer_sizes — adjusting the number of hidden layers and neurons per layer helps the model learn more complex patterns or prevents overfitting. (2) learning_rate_init — the initial learning rate controls the step size during weight updates; a well-tuned rate can speed up convergence and improve accuracy. (3) max_iter — increasing the maximum number of iterations gives the optimizer more epochs to converge, especially helpful when the model has not yet reached an optimal solution. Additional hyperparameters worth tuning include activation (e.g., relu, tanh), alpha (L2 regularization strength), and solver (e.g., adam, sgd).
